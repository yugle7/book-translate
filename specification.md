# Спецификация для AI Coding Агента: Веб-приложение "Бок о бок: оригинал и перевод"

## 1. Цель разработки

Создать одностраничное веб-приложение, которое позволяет пользователю отображать оригинальный текст и его перевод в двух колонках (side‑by‑side) с соблюдением следующих условий:

- Абзацы оригинала и перевода визуально выровнены так, чтобы начало каждого i‑го абзаца в левой и правой колонке находилось на одной горизонтальной линии.
- Между колонками присутствует вертикальный разделитель (gutter), который пользователь может перетаскивать мышью влево/вправо, изменяя ширину каждой колонки.
- При перетаскивании разделителя или изменении размера окна браузера выравнивание абзацев сохраняется.

## 2. Функциональные требования

### 2.2. Разбиение на абзацы

- Текст из каждого текстового поля разбивается на абзацы по правилу: **два или более последовательных символа перевода строки** (`\n\n` или `\r\n\r\n`). Одиночные переносы строк внутри абзаца сохраняются как часть абзаца (переносятся в HTML с помощью CSS `white-space: pre-wrap;`).
- Если количество абзацев в оригинале и переводе различается, недостающие абзацы считаются **пустыми строками** (отображается пустой блок с минимальной высотой).

### 2.3. Отображение колонок

- Левая колонка – абзацы оригинала (каждый в своём контейнере, например `<div class="paragraph">`).
- Правая колонка – абзацы перевода (аналогичная структура).
- Каждая колонка имеет независимую вертикальную прокрутку (при переполнении по высоте).

### 2.4. Вертикальное выравнивание абзацев (одна строка)

- Для каждой пары абзацев (i‑й абзац оригинала и i‑й абзац перевода) их **начало** (верхняя граница) должно находиться на одном уровне по вертикали.
- Технически это достигается вычислением высоты каждого блока абзаца и установкой для обоих блоков **одинаковой высоты**, равной `max(высота_оригинала, высота_перевода)`.
- Синхронизация высот должна выполняться:
  - после загрузки/изменения текста,
  - после каждого движения разделителя,
  - после изменения размера окна браузера.
- Для корректной работы необходимо использовать JavaScript (без сторонних библиотек или с минимальными зависимостями).

### 2.5. Вертикальный разделитель

- Между колонками располагается узкая полоса (ширина 6–8 пикселей), которую можно захватить мышью и перемещать горизонтально.
- При перетаскивании разделителя левая и правая колонки изменяют свою ширину (пропорционально или абсолютно). Например: изначально колонки занимают по 50% ширины родительского контейнера; при смещении разделителя ширина левой колонки становится `X`, правой – `(100% - X - ширина_разделителя)`.
- При перетаскивании не должно происходить перерисовки абзацев (только изменение ширины), но **синхронизация высот должна запускаться по окончании движения** (для надёжности можно использовать `requestAnimationFrame` во время движения и финальный пересчёт).

### 2.6. Автоматическое обновление

- При изменении содержимого любого текстового поля (событие `input`) приложение автоматически переразбивает текст на абзацы, перестраивает DOM обеих колонок и выполняет синхронизацию высот.
- При загрузке файла – аналогично.

## 3. Нефункциональные требования

### 3.1. Производительность

- Синхронизация высот для большого числа абзацев (до 100) не должна вызывать заметных фризов. Использовать `requestAnimationFrame` и избегать принудительных синхронных перерасчётов в циклах.

### 3.2. Кроссбраузерность

- Поддержка последних версий Chrome, Firefox, Safari, Edge.

### 3.3. Адаптивность

- При ширине окна менее 768px разделитель должен исчезать, а колонки отображаться друг под другом (оригинал сверху, перевод снизу). На мобильных устройствах выравнивание абзацев в таком режиме не требуется (или можно оставить как есть).

### 3.4. Стилизация и UX

- Чёткие границы между колонками, удобный захват разделителя (курсор `col-resize`).
- Абзацы должны быть читаемыми: шрифт без засечек (sans-serif), размер 16px, межстрочный интервал 1.5, отступы между абзацами 1em.
- Каждая колонка должна иметь собственную тень или фон для лёгкого визуального разделения (например, очень светлый серый фон у левой, белый у правой).

## 4. Техническая реализация (рекомендации для AI)

### 4.1. Общая архитектура

- **HTML** – один файл `index.html`.
- **CSS** – встроенный в `<style>` или отдельный файл.
- **JavaScript** – встроенный или отдельный, без внешних библиотек (vanilla JS).

### 4.2. Структура DOM (основные элементы)

```html
<div class="app">
  <div class="input-panel">
    <div class="input-group">
      <label>Оригинал</label>
      <textarea id="sourceText"></textarea>
      <input type="file" id="sourceFile" accept=".txt" />
    </div>
    <div class="input-group">
      <label>Перевод</label>
      <textarea id="targetText"></textarea>
      <input type="file" id="targetFile" accept=".txt" />
    </div>
  </div>
  <div class="split-view" id="splitView">
    <div class="pane left" id="leftPane">
      <div id="leftParagraphs"></div>
    </div>
    <div class="gutter" id="gutter"></div>
    <div class="pane right" id="rightPane">
      <div id="rightParagraphs"></div>
    </div>
  </div>
</div>
```

### 4.3. Разделитель (ресайз колонок)

- Родитель `.split-view` – `display: flex; width: 100%; height: calc(100vh - 200px);` (высота подбирается под оставшееся пространство).
- `.pane` – `flex: 1 1 0; overflow-y: auto;`.
- `.gutter` – `width: 8px; background: #aaa; cursor: col-resize;`.
- При перетаскивании:
  - вычисляется новая ширина левой панели в пикселях относительно родителя,
  - для левой панели задаётся `flex: 0 0 <ширина>px`,
  - для правой панели `flex: 1 1 auto` (или тоже фиксированная ширина).
- Альтернативно можно использовать `position: absolute` с событием `mousemove`, но проще через `flex`.

### 4.4. Разбиение текста на абзацы

```javascript
function splitIntoParagraphs(text) {
  // Разделитель: две или более пустых строк
  return text.split(/\n\s*\n/).map((p) => (p.trim() === "" ? "" : p));
  // Важно: сохранять пробелы внутри абзаца – при отображении использовать CSS white-space: pre-wrap;
}
```

### 4.5. Создание DOM для абзацев

- Функция `renderParagraphs(container, paragraphsArray)` создаёт внутри контейнера `#leftParagraphs` и `#rightParagraphs` элементы `<div class="paragraph">` с текстом (если абзац пустой – добавить класс `empty` и минимальную высоту 1em).
- Вызывается при каждом обновлении текста.

### 4.6. Синхронизация высот

- Функция `syncHeights()`:
  1. Получает все элементы `.paragraph` из левой и правой колонок (их количество должно быть одинаковым; если нет – дополнить пустыми).
  2. Для каждого индекса `i`:
     - `const leftHeight = leftParas[i].scrollHeight;`
     - `const rightHeight = rightParas[i].scrollHeight;`
     - `const maxHeight = Math.max(leftHeight, rightHeight);`
     - `leftParas[i].style.height = maxHeight + 'px';`
     - `rightParas[i].style.height = maxHeight + 'px';`
- Важно: перед измерением высоты необходимо временно убрать ограничения по высоте (если были заданы ранее) – установить `height: auto`, измерить, затем задать общую высоту.
- Вызывать `syncHeights()` после рендеринга, после изменения размеров окна и после завершения перетаскивания разделителя.
- Для отложенного вызова при ресайзе использовать `debounce` или `ResizeObserver`.

### 4.7. Обработчики событий

- Текстовые поля – `addEventListener('input', onTextChange)`.
- Загрузка файлов – `addEventListener('change', onFileLoad)`.
- Разделитель – `mousedown`, `mousemove`, `mouseup` на глобальном уровне.
- Изменение размера окна – `window.addEventListener('resize', () => { syncHeights(); })`.

### 4.8. Примерный алгоритм обновления при изменении текста

```javascript
function updateTexts() {
  const source = document.getElementById("sourceText").value;
  const target = document.getElementById("targetText").value;
  const sourceParas = splitIntoParagraphs(source);
  const targetParas = splitIntoParagraphs(target);
  const maxLen = Math.max(sourceParas.length, targetParas.length);
  // Дополняем массивы пустыми строками
  while (sourceParas.length < maxLen) sourceParas.push("");
  while (targetParas.length < maxLen) targetParas.push("");

  renderParagraphs(document.getElementById("leftParagraphs"), sourceParas);
  renderParagraphs(document.getElementById("rightParagraphs"), targetParas);

  // После того как DOM обновлён, синхронизируем высоты
  syncHeights();
}
```

## 5. Пример демонстрационных данных по умолчанию

**Оригинал (русский):**

```
Всем привет! Это пример приложения для синхронного отображения текста и перевода.

Здесь второй абзац. Он чуть длиннее, чтобы показать, как выравниваются блоки разной высоты.

Третий абзац — короткий.
```

**Перевод (английский):**

```
Hello everyone! This is an example of an app for side‑by‑side text and translation.

Here is the second paragraph. It is a bit longer to demonstrate how blocks of different heights are aligned.

The third paragraph is short.
```

## 6. Критерии приёмки для AI агента

- Приложение открывается в браузере и корректно отображает две колонки с текстом.
- Абзацы выровнены по начальной строке даже при разной высоте.
- Разделитель перетаскивается и меняет ширину колонок; после перетаскивания выравнивание не ломается.
- Загрузка из файлов работает (поддерживаются `.txt` в кодировке UTF-8).
- При изменении содержимого текстовых полей интерфейс обновляется автоматически.
- Код чист, закомментирован в ключевых местах, не вызывает ошибок в консоли.

## 7. Дополнительные рекомендации

- Для плавности перетаскивания разделителя можно добавить минимальную ширину колонок (например, 150px).
- При очень большом количестве абзацев (100+) стоит использовать виртуализацию, но в рамках данной спецификации это не обязательно.
- Чтобы избежать "мерцания" при синхронизации высот, можно установить `.paragraph` с `transition: height 0.05s ease-out;`.

## 8. Ожидаемый результат

AI Coding Агент должен сгенерировать **один HTML-файл** (или несколько: HTML, CSS, JS), который полностью реализует описанное поведение. Код должен быть самодостаточным, готовым к запуску в любой современной среде (локальный файл или веб-сервер).
